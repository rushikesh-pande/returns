# Return Service

Return Service for managing order returns, refunds, and customer returns lifecycle.

## Features

- ✅ Initiate return requests
- ✅ Approve/Reject returns
- ✅ Generate shipping labels
- ✅ Track return status
- ✅ Process refunds automatically
- ✅ Kafka event publishing
- ✅ Customer return history

## API Endpoints

### Return Management

#### Initiate Return
```http
POST /api/v1/returns/initiate
Content-Type: application/json

{
  "orderId": "ORD-12345",
  "customerId": "CUST-001",
  "customerEmail": "customer@example.com",
  "reason": "DEFECTIVE_PRODUCT",
  "reasonDescription": "Product arrived damaged",
  "refundAmount": 99.99,
  "isPickupRequired": true,
  "pickupAddress": "123 Main St, City, State 12345"
}
```

#### Approve Return
```http
PUT /api/v1/returns/{returnId}/approve
```

#### Reject Return
```http
PUT /api/v1/returns/{returnId}/reject?reason=Out of return window
```

#### Mark as Received
```http
PUT /api/v1/returns/{returnId}/received
```

#### Process Refund
```http
POST /api/v1/returns/{returnId}/refund
```

#### Get Return Status
```http
GET /api/v1/returns/{returnId}
```

#### Get Customer Returns
```http
GET /api/v1/returns/customer/{customerId}
```

#### Get Returns by Status
```http
GET /api/v1/returns/status/APPROVED
```

## Return Status Flow

1. **REQUESTED** - Customer initiates return
2. **APPROVED** - Return request approved
3. **REJECTED** - Return request rejected
4. **SHIPPING_LABEL_GENERATED** - Shipping label created
5. **IN_TRANSIT** - Product being returned
6. **RECEIVED** - Product received back
7. **INSPECTING** - Quality inspection in progress
8. **INSPECTION_PASSED** - Product passed inspection
9. **INSPECTION_FAILED** - Product failed inspection
10. **REFUND_INITIATED** - Refund process started
11. **REFUNDED** - Refund completed
12. **COMPLETED** - Return process completed
13. **CANCELLED** - Return cancelled

## Database Schema

```sql
CREATE TABLE returns (
    id BIGSERIAL PRIMARY KEY,
    return_id VARCHAR(255) UNIQUE NOT NULL,
    order_id VARCHAR(255) NOT NULL,
    customer_id VARCHAR(255) NOT NULL,
    customer_email VARCHAR(255) NOT NULL,
    reason VARCHAR(50) NOT NULL,
    reason_description VARCHAR(1000),
    status VARCHAR(50) NOT NULL,
    refund_amount DECIMAL(10,2) NOT NULL,
    shipping_label_url VARCHAR(500),
    tracking_number VARCHAR(255),
    requested_date TIMESTAMP NOT NULL,
    approved_date TIMESTAMP,
    received_date TIMESTAMP,
    refunded_date TIMESTAMP,
    processing_notes VARCHAR(1000),
    is_pickup_required BOOLEAN NOT NULL,
    pickup_address VARCHAR(500)
);
```
